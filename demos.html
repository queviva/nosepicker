so you think you need a more complicated colorpicker? fine then ...

nosepicker.js is totally complicated, and bloated, and ridiculous, and allows for all kinds of customizations - it's the perfect thing for you picky nosepickers
include the script, along with any options in the data-param; specify nosepicker elements with a data-param; handle any events with listeners, as usual; set any options dynamically by dispatching an event to the nosepicker: 

<script src="nosepicker.js"
 data-nose='{
    "selector"  : "nose",
    "kind"      : "touch,wheel", // wheel|touch|swipe|coast|point
    "colorsrc"  : "background-color",
    "loaded"    : "nose-loaded",
    "input"     : "nose-input",
    "colorSelf" : true,
    "colorText" : true,
    "pattern"   : true,
    "able"      : true,
    "makeRef"   : false,
    "sens"      : { X: 1, Y: 1 }
 '}
></script>

<div id="picker" data-nose></div>

<script>
    
    picker.addEventListener('nose-input', e => {
    
        // e.detail.value is an hsla color string
        
        // e.detail.hsla is an object ...
        // e.detail.hsla.h is the hue
        // e.detail.hsla.s is the saturation
        // e.detail.hsla.l is the luminosity
        // e.detail.hsla.a is the opacity
        
        // e.detail.vals is an array ...
        // e.detail.vals[0] is the hue
        // e.detail.vals[1] is the saturation
        // e.detail.vals[2] is the luminosity
        // e.detail.vals[3] is the opacity
        
    });
    
    // set options dynamically by dispatching events ...
    // settable options are value|kind|able|sens
    
    picker.dispatchEvent(new CustomEvent(
        'nose-set-value', {detail:'#f00'})
    );
    
    picker.dispatchEvent(new CustomEvent(
        'nose-set-kind', {detail: 'coast,point'})
    );
    
    picker.dispatchEvent(new CustomEvent(
        'nose-set-able', {detail: false})
    );
    
    picker.dispatchEvent(new CustomEvent(
        'nose-set-sens', {detail: {X:1, Y:1})
    );
    
</script>
now I bet you want to know what all these things do, don't you? ok, here it is:
  • selector: as per yoosh, this is the string that you want to be the data-param and nose-input event; by default, nosepicker.js selects all elements with a data-nose parameter, and generates a nose-input event; if, for example, you changed this to "picker", then the script would select all elements with data-picker as a parameter, and generate a picker-input event

  • kind: this is a coma-separated string which lists all the kinds of nosepicker listeners you want to add to just this specific element; the default is "touch,wheel" which will create both a scroll listener and a touch listener for touchscreens; the possible options are:

    • wheel - scrolling up|dn, left|right; ctrl+scroll changes transparency and shift+scroll changes sensitivity
    • touch - moving a touchpoint on touchscreens changes hue and luminosity, using two touchpoints changes transparency and saturation,
    • swipe - either on a touchscreen, or with a mouse, just swipe around; again, two fingers acts as ctrl+swipe
    • coast - this is point-and-fly type of navigation; you just click in one spot, either on a touchscreen on with a mouse, and then you will fly through the colors continuously in whichever direction you're pointing
    • point - is just like coast but uses a pointer event

  • colorsrc: the style aspect that nosepicker.js will use as the starting color; this might be "fill" if you have, for example, an svg path

  • loaded: the name of the nose-loaded event that gets generated when a nosepicker loads

  • input: the name of the nose-input event that gets generated when the color changes

  • colorSelf: boolean as to whether or not the picker should color its own self - by default it will, but you might not want that

  • colorText: true|false if you want the color of the text inside the nosepicker to be contrasted with the selected color

  • pattern: boolean true|false if you want a patterned background behind transparent colors

  • able: simply a boolean as to whether the nosepicker is enabled

  • sens: adjusts the sensitivity of the motion in the X, and Y, direction - this is an object specifying a float for the adjustment to both the X‑sensitivity and the Y‑sensitivity

all interactions with nosepickers are done through dispatching events; various options can be set directly, by sending event.detail values to the nosepicker; the prefix will be whatever the selector is, which by default is nose‑
  • nose-set-value: this sets the color value to whatever valid css color string gets passed in

  • nose-set-kind: you can actually change the kind of nosepicker while it's being used; you could dynamically switch from a scroll listener to a coast listener

  • nose-set-able: passing a boolean will either disable or enable to nosepicker

  • nose-set-sens: again, this is an object that has an X, and a Y, float value adjustment to the nosepicker's sensitivity; for example, to double the sensitivity, pass in {X:0.5, Y:0.5}

the detail of the nose-loaded event is a function that allows a shorthand method for frequent communication through dispatch; if you add a listener for window.nose-loaded you can create a function that allows communication with nose-pickers like this:
patchooli('picker', 'nose-set-value', '#c30')

the parameters of that function are:
  • a picker object, or a string of its ID
  • the string name of the event
  • the value to pass to the picker
the sample nosepickers above will reset to their original colors when you double-click them - as another example, if you wanted to directly set the color of a nosepicker called 'picker' to a hex value for red when you double-click the window, this listener would do that: 

            
window.addEventListener('dblclick', () => {
    picker.dispatchEvent(new CustomEvent(
        'nose-set-value',
        {detail: '#c30'}
    ));
});
